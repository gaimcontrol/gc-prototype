How this works. Each per-flow demo is hand-coded HTML/CSS, no React, no build step. Every screen is wired straight to the matching screen ID in the spec (ON1, BK1, CC1, etc.) so engineers and reviewers can compare side-by-side. Decisions from the log are applied inline. When all flows for a device are signed off, the corresponding full prototype at the top of this page stitches them together with cross-flow navigation. Bottom tabs on mobile (Home / Banking / Anna / Activity / Profile), sidenav on web. The full prototype is the artefact you walk clients through; the per-flow demos are the units you build and review.
